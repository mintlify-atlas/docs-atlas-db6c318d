Get both microservices up and running in your local environment.

## Prerequisites

Before you begin, ensure you have the following installed:

| Tool | Minimum Version |
|------|----------------|
| Python | 3.10+ |
| pip | Latest |
| Node.js | 18+ |
| npm | 9+ |
| Docker & Docker Compose | 20+ / 2.0+ (optional) |

## Local Development Setup

<Tabs>
  <Tab title="Python Service">
    <Steps>
      <Step title="Navigate to the Python service directory">
        ```bash
        cd python-service
        ```
      </Step>
      
      <Step title="Create and activate a virtual environment">
        <CodeGroup>
          ```bash macOS/Linux
          python -m venv venv
          source venv/bin/activate
          ```
          
          ```bash Windows
          python -m venv venv
          venv\Scripts\activate
          ```
        </CodeGroup>
      </Step>
      
      <Step title="Install dependencies">
        ```bash
        pip install -r requirements.txt
        ```
      </Step>
      
      <Step title="Run the server">
        ```bash
        python run.py
        ```
        
        The Python service will start on **http://localhost:5000**
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="Node.js Service">
    <Steps>
      <Step title="Navigate to the Node service directory">
        ```bash
        cd node-service
        ```
      </Step>
      
      <Step title="Install dependencies">
        <CodeGroup>
          ```bash npm
          npm install
          ```
          
          ```bash yarn
          yarn install
          ```
          
          ```bash pnpm
          pnpm install
          ```
        </CodeGroup>
      </Step>
      
      <Step title="Run the server">
        ```bash
        npm start
        ```
        
        The Node.js service will start on **http://localhost:3000**
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="Docker (Full Stack)">
    <Steps>
      <Step title="Start all services with Docker Compose">
        ```bash
        docker-compose up --build
        ```
        
        This will start:
        - Python API: **http://localhost:5000**
        - Node.js API: **http://localhost:3000**
        - PostgreSQL: **localhost:5432**
        - Redis: **localhost:6379**
      </Step>
      
      <Step title="Verify services are running">
        Check the Node.js health endpoint:
        
        ```bash
        curl http://localhost:3000/api/health
        ```
        
        And verify Python service is running by accessing any endpoint:
        
        ```bash
        curl http://localhost:5000/api/products
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Run the Tests

Verify your setup by running the test suites:

<Tabs>
  <Tab title="Python Tests">
    ```bash
    cd python-service
    source venv/bin/activate  # macOS/Linux
    python -m pytest tests/ -v
    ```
  </Tab>
  
  <Tab title="Node.js Tests">
    ```bash
    cd node-service
    npm test
    ```
  </Tab>
  
  <Tab title="All Tests">
    ```bash
    chmod +x scripts/run_all_tests.sh
    ./scripts/run_all_tests.sh
    ```
  </Tab>
</Tabs>

## Make Your First API Call

<Steps>
  <Step title="Register a new user">
    ```bash
    curl -X POST http://localhost:5000/api/auth/register \
      -H "Content-Type: application/json" \
      -d '{
        "email": "user@example.com",
        "password": "securepassword123",
        "name": "John Doe"
      }'
    ```
    
    Response:
    ```json
    {
      "message": "User registered successfully",
      "user": {
        "id": 1,
        "email": "user@example.com",
        "name": "John Doe",
        "role": "customer"
      },
      "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    }
    ```
  </Step>
  
  <Step title="Login and get JWT token">
    ```bash
    curl -X POST http://localhost:5000/api/auth/login \
      -H "Content-Type: application/json" \
      -d '{
        "email": "user@example.com",
        "password": "securepassword123"
      }'
    ```
    
    Save the `access_token` from the response for authenticated requests.
  </Step>
  
  <Step title="Create a product (authenticated)">
    ```bash
    curl -X POST http://localhost:5000/api/products \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
      -d '{
        "name": "Wireless Headphones",
        "description": "Premium noise-cancelling headphones",
        "price": 299.99,
        "stock": 50,
        "category": "electronics"
      }'
    ```
  </Step>
  
  <Step title="List all products (public endpoint)">
    ```bash
    curl http://localhost:5000/api/products
    ```
    
    Response:
    ```json
    {
      "products": [
        {
          "id": 1,
          "name": "Wireless Headphones",
          "description": "Premium noise-cancelling headphones",
          "price": 299.99,
          "stock": 50,
          "category": "electronics"
        }
      ],
      "pagination": {
        "page": 1,
        "per_page": 20,
        "total": 1,
        "pages": 1
      }
    }
    ```
  </Step>
</Steps>

## Environment Configuration

Both services use environment variables for configuration. For local development, the default values work out of the box.

<Tabs>
  <Tab title="Python Service">
    Create a `.env` file in `python-service/`:
    
    ```bash
    FLASK_ENV=development
    SECRET_KEY=dev-secret-key
    DATABASE_HOST=localhost
    DATABASE_PORT=5432
    ```
  </Tab>
  
  <Tab title="Node.js Service">
    Create a `.env` file in `node-service/`:
    
    ```bash
    NODE_ENV=development
    PORT=3000
    JWT_SECRET=your-secret-key
    DB_HOST=localhost
    DB_PORT=5432
    ```
  </Tab>
</Tabs>

<Note>
For testing, both services automatically use SQLite in-memory databases, so you don't need PostgreSQL running locally.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="psycopg2-binary fails to install">
    This is expected for local development. The Python service uses SQLite for testing, so PostgreSQL dependencies are optional.
    
    Skip the error or install without PostgreSQL support:
    ```bash
    pip install -r requirements.txt --no-deps
    ```
  </Accordion>
  
  <Accordion title="ModuleNotFoundError: No module named 'app'">
    Make sure you're running commands from the `python-service/` directory, not the project root.
  </Accordion>
  
  <Accordion title="Port already in use">
    If port 5000 or 3000 is already in use, you can specify a different port:
    
    ```bash
    # Python
    FLASK_RUN_PORT=5001 python run.py
    
    # Node.js
    PORT=3001 npm start
    ```
  </Accordion>
  
  <Accordion title="sqlite3 build errors (Node)">
    If you encounter native module build errors for sqlite3:
    
    ```bash
    npm rebuild sqlite3
    ```
    
    Or ensure you have native build tools installed (python, make, gcc).
  </Accordion>
  
  <Accordion title="Docker containers fail to start">
    Check if ports are available and no other containers are running:
    
    ```bash
    docker-compose down
    docker-compose up --build
    ```
  </Accordion>
</AccordionGroup>
